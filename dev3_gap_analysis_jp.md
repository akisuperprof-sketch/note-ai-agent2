# Note AI Agent 2 - 仕様差異分析と対策リスト

**作成日:** 2025-12-30
**分析元:** GPT構築チャット履歴 vs 現在の仕様書 ()
**分析者:** Dev3 Verifier

## 1. 差異分析 (Gap Analysis)

チャット履歴の「アクシオム」による提案・評価と、現在の仕様書を比較し、取り込むべき要素を特定しました。

### A. AIの能動性・パートナーシップ (UX/Product Gap)
*   **チャット提案:** 「AIおすすめモード」や「判断代行」機能。ユーザーが迷わずに済むよう、AIが推奨設定を提示する機能。
*   **現在仕様:** ユーザーが細かくパラメータ（文字数、トーン、読者、目的など）を入力する「高機能フォーム」型。
*   **Gap:** 「能動的なパートナー」としての機能が不足している。ツール止まりになっている。

### B. 生成後の価値最大化 (LTV Gap)
*   **チャット提案:** 生成完了後の「次の一手」提示（X投稿作成、有料note化アドバイス、別切り口での再生成など）。
*   **現在仕様:** 記事生成と画像生成でプロセスが終了（displaying result）。
*   **Gap:** ユーザーの収益化行動への導線が途切れている。

### C. UI/状態遷移の深化 (UI/UX Gap)
*   **チャット提案:** より具体的な状態遷移（Phase0:入力正規化 -> Phase1:構成 -> Phase2:執筆 -> Phase3:編集 -> ...）。「編集者は一度推敲する」というメタファーの実装。
*   **現在仕様:** 大枠のステップ（構成作成、執筆、画像生成）はあるが、内部的な「推敲・編集」フェーズが明示的でない。

### D. プロンプト最適化 (Quality Gap)
*   **チャット提案:** 画像プロンプト生成時に、「記事全体を一文で要約したテーマ要約」を抽出し、それを画像プロンプトに注入するロジック。
*   **現在仕様:** 記事の冒頭800文字をそのまま渡して要約させている可能性あり（API実装依存）。
*   **Gap:** 「テーマ要約」という中間生成物を噛ませることで、画像のブレを減らせる可能性がある。

## 2. 対策リスト (Action Plan)

これらを実装に組み込むための具体的なタスクリストです。

### 優先度：高 (High Priority) - 即座に実装すべき項目
1.  **AI判断代行（おすすめ設定）の実装**
    *   **仕様:** 入力フォームに「おまかせ」ボタン、またはテーマ入力時に自動で推奨パラメータ（文字数・トーン）を提案するロジックを追加。
    *   **理由:** 初心者の離脱を防ぎ、「AIエージェント感」を高めるため。

2.  **生成プロセスの微調整（要約フェーズの追加）**
    *   **仕様:** 画像生成の前に、明示的に「記事のテーマを抽象的な英語1文で要約する」ステップを追加し、それを画像生成プロンプトに使用する。
    *   **理由:** 画像生成の品質（特に抽象度と統一感）を安定させるため。

3.  **完了画面のアクション強化**
    *   **仕様:** 結果画面の下部に「Xで告知する（投稿文作成）」ボタンや「有料版のアイデアを聞く」ボタンを追加（モックでも可）。
    *   **理由:** ユーザーの体験を「書く」から「発信する/稼ぐ」へ拡張するため。

### 優先度：中 (Medium Priority) - 次のステップで検討
4.  **「編集・推敲」フェーズの可視化**
    *   **仕様:** 生成ログに「文章を整えています（推敲中）...」というステップを追加し、内部的にも推敲プロンプトを一回通す。
    *   **理由:** 文章品質の向上と、ユーザーへの安心感提供。
